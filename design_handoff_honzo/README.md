# Handoff：本草 HONZŌ — 知識研究 / RAG 平台前端

## Overview（概述）
本草 HONZŌ 是一套「知識研究 / RAG（檢索增強生成）平台」的後台介面設計，共 **7 個畫面**：典藏、問答、收錄、用量、收錄條目、解析流程設定、權限。核心能力是把文件收錄後拆解成向量與知識圖譜，供 AI 問答時檢索、引用並標注來源。本交接包是要在你的新產品中，以此設計為視覺與結構基底重新實作。

> 命名說明：「本草 HONZŌ」為 placeholder 品牌名，導覽用詞（典藏 / 問答 / 收錄…）亦可替換。實作時請以你的實際品牌與資訊架構為準，但**版面、配色、字體、元件結構**照此設計。

## About the Design Files（關於這些設計檔）
本包中的 `.dc.html` 檔是**以 HTML 製作的設計參考稿（高擬真原型）**，用來呈現「應該長什麼樣、有哪些元件與狀態」，**並非可直接上線的生產程式**。所有資料都是寫死的假資料、互動僅為視覺呈現、`.dc.html` 是設計工具專用格式。

你的任務是**在目標程式環境中重建這些畫面**（React / Vue / Svelte / 原生皆可），沿用該環境既有的元件庫與慣例；若尚無環境，請為此產品選擇最合適的框架再實作。把寫死的資料改為 API 取得的 state、把靜態按鈕綁上事件邏輯即可。

## Fidelity（擬真度）
**高擬真（hifi）**。顏色、字體、間距、元件狀態皆為最終值，請盡量像素級還原。互動流程為示意，需你補上真實邏輯。

## Screens / Views（畫面規格）

### 共用 — 頂部導覽（所有頁面一致）
- **第一列**（padding 16px 40px）：左為方形 logo（34px，`#A65A35` 底、明體「本」字）+ 品牌「本草 / HONZŌ」+ 分隔線 + 副標「知識研究平台」；右為搜尋框（1px 邊框、`/` 快捷鍵提示）+「✦ 智詢」深色按鈕 + 語言 + 使用者方形頭像（`#6B5640`、mono 縮寫）。
- **第二列**（tab 列，下方 1px 分隔線）：橫向文字分頁「總覽 / 典藏 / 圖譜 / 問答 / 收錄 / 解析 / 用量 / 設定」，**當前分頁** `#211C16` 字重 500 + 下方 2px `#A65A35` 底線，其餘 `#8A7D67`。右端有 mono 小標（如「生物科 · BIOLOGY」）。
- **無左側欄**（這是刻意的差異化決定）。

### 01 · 典藏（Library）— `01-Knowledge-List.dc.html`
- **用途**：瀏覽已收錄的知識條目，依學門分組。
- **版面**：頂部導覽 + 內容區（padding 38px 56px）。標題列（明體「典藏」42px + mono 上標 + 統計副標 + 右側「圖譜檢視」「＋ 收錄條目」按鈕）。下方 pill 篩選列（全部 / 各學門計數）。
- **主體**：依學門（細胞學 / 遺傳學 / 演化學）分組，每組有標題列（明體 21px + mono 英文 + 條目數，下 1.5px 黑線），組內每列為 grid `46px 1fr 130px 92px 120px`：序號（mono、弁柄色）/ 明體條目名（+「探討活動」黃土色小標籤）/ token 數 / 狀態標籤 / 引用數·時間。列 hover 底色 `#FBF8F0`。
- **資料**：16 條生物學條目（細胞學 8、遺傳學 5、演化學 3）。

### 02 · 問答（RAG Chat）— `02-Chat.dc.html`
- **用途**：與知識助理對話，答案僅依典藏知識生成並標注引用來源。
- **版面**：頂部導覽 + 主體分兩欄。
  - **左（對話串，flex:1）**：串標題列（AI 方形圖示 + 對話名 + 域別 + 右側「HYBRID · VECTOR + GRAPH」標記）；訊息區為聊天氣泡 —— 使用者訊息靠右（`#6B5640` 底白字），AI 訊息靠左（`#FBF8F0` 卡片 + 1px 邊框，內含答案 + 底部「引用來源」區塊：來源 chip = 序號 + 明體條目名 + 相關度分數 `#7E8060`）；底部 composer（1.5px 黑框輸入 + 深色「發送」+ mono 提示）。
  - **右（可開合 session 面板，296px）**：頂部「SESSIONS · 對話」+ 收合鈕「»」；「＋ 新對話」邊框按鈕；依時間分組（今日 / 本週）的對話清單，當前項深色底、其餘 hover `#F1EBDD`，每項含標題 + 則數·時間；底部「N 個對話 / 清除全部」。
- **實作重點**：初始為「提問起點（空狀態，置中大標『今天想了解什麼?』+ 建議問題）」，送出第一則後**變形為此對話串版面** — 這是關鍵互動。

### 03 · 收錄（Ingestion）— `03-Ingestion.dc.html`
- **用途**：監看文件收錄作業在 RAG 管線中的進度。
- **版面**：標題列（明體「收錄管線」30px + 右側吞吐量）。下方 **Notion 式 filter 列**（漏斗「篩選」鈕 + 可移除的條件 chip「管線 是 GRAPH ×」「狀態 是 進行中 ×」+「＋ 新增篩選」虛線鈕 + 右端「排序」「25/每頁」）。
- **主體**：**五階段看板**（grid 5 欄）：佇列 → 切塊 → 向量化 → 圖譜抽取 → 完成。每欄標題（明體 + 計數，下 1.5px 黑線 + mono 英文）；欄內作業卡（明體條目名 + 管線標籤 + 進度條 `#A65A35` / 狀態）。
- **資料**：9 筆作業，狀態分佈於各階段。

### 04 · 用量（Usage / Analytics）— `04-Quota.dc.html`
- **用途**：檢視團隊 token 用量與趨勢。
- **版面**：頂部標題列（明體「用量分析」30px + 週期）+ **右側總額大數字**（mono 44px `931,974` / `10,000,000` TOK + 9% 進度條）。下方 grid `1.5fr 1fr`（兩列）：
  - 左（跨兩列）：**趨勢面積圖**（近 30 天，SVG，GRAPH/VECTOR/CHAT 三線 + 面積填充 + 圖例 + 日期軸）。
  - 右上：**管線拆解長條圖**（GRAPH 512K / VECTOR 328K / CHAT 92K）。
  - 右下：**成員用量表**（方形頭像 + 名 + token 數）。

### 05 · 收錄條目（Create Entry）— `05-Create.dc.html`
- **用途**：新增一筆知識條目並上傳來源檔。
- **版面**：**置中單欄書院式表單**（max-width 880px），每列 grid `200px 1fr`：左為欄位標題（明體 + mono 代號 + 說明），右為輸入。欄位：標題（明體輸入框）/ 歸屬學門（分段按鈕：細胞學選中深色）/ 可閱覽範圍（公開·教師·學生·客座 × 基本/限定 的勾選矩陣，`✓` `#7E8060`）/ 來源檔案（虛線拖放區，↥ 圖示 + 格式說明）。底部「取消 / 儲存草稿」。

### 06 · 解析流程設定（Extraction Flow）— `06-Extraction.dc.html`
- **用途**：設定 RAG 收錄的解析管線參數。
- **版面**：標題「解析流程設定」30px。下方**五步驟垂直流程**（左側編號圓點 + 連接線，弁柄色標示重點步驟）：1 領域模板（生物學 v1 選中，含通用/醫學/化學/課綱對齊）→ 2 切塊策略（語意/固定，800/重疊120）→ 3 嵌入模型（text-embedding-3-large 3072d）→ 4 實體抽取 Schema（構造/分子/過程色點標籤 + ＋新增類型）→ 5 關係抽取（1-hop、信賴度 0.65、12 種述詞）。底部動作列（說明「變更後既有知識標記為待重新解析」+ 還原預設 / 儲存並重新解析）。

### 07 · 權限（Roles / Matrix）— `07-Roles.dc.html`
- **用途**：檢視角色 × 能力的權限分佈。
- **版面**：標題「權限矩陣」30px。下方**矩陣表格**（grid `1.4fr 1fr 1fr 1fr 1fr 1.4fr 90px`）：欄為 讀/寫/審/管 四能力 + 範圍 + 授權數；列為 主編（ADMIN）/ 編修 / 審訂 / 閱覽。格內方塊：`#A65A35`=管理員授權、`#6B5640`=顯式授權、1.5px 空心邊框=無此能力。底部圖例。

## Interactions & Behavior（互動與行為）
- **02 問答的空→對話變形**：初始置中提問起點，送出後轉為左對話串 + 右 session 面板。
- **02 session 面板**：可透過「»」收合/展開；點對話項切換內容；「＋ 新對話」清空為空狀態。
- **03 filter**：Notion 式 —— 點「篩選」加條件、每個 chip 可點「×」移除、可多條件疊加、「排序」切換排序。
- **01 條目列 / 07 角色列**：hover 變底色，點擊進入詳情（需你補頁）。
- **05 表單**：分段按鈕單選、勾選矩陣多選、拖放上傳 + 格式/大小驗證（pdf/docx/md/txt，≤50MB）。
- 所有「發送 / 儲存 / 採用」按鈕需綁後端呼叫。

## State Management（狀態）
- **01**：條目清單（依學門分組）、篩選 tab、排序。
- **02**：對話 sessions[]、當前 session、messages[]（role/text/citations[]）、面板開合狀態、輸入草稿；送出時呼叫檢索+生成 API 回傳答案與引用來源。
- **03**：作業 jobs[]（含階段/管線/進度）、filter 條件[]、排序；建議以輪詢或 WebSocket 更新進度。
- **04**：時間區間、趨勢資料、管線拆解、成員用量。
- **05**：表單欄位、上傳檔案、解析選項；提交建立條目 + 觸發收錄作業。
- **06**：解析設定檔（各步驟參數）、當前模板；儲存後標記受影響知識為待重新解析。
- **07**：角色[]（能力 × 範圍組合）；編輯權限。

## Data Fetching（資料需求，示例）
- `GET /entries`（01：id, title, domain, status, tokens, citations, updatedAt）
- `POST /chat`（02：query → answer, sources[{entryId, title, score, retrievalType}]）+ `GET /sessions`
- `GET /ingestion/jobs`（03：id, entryTitle, pipeline, stage, progress, tokens）
- `GET /usage`（04：total, limit, trend[], byPipeline[], byMember[]）
- `POST /entries` + `POST /ingestion`（05）
- `GET/PUT /extraction-profile`（06）
- `GET/PUT /roles`（07）
